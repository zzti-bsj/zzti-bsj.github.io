创建文章：在hexo文件夹下， hexo new “文章标题”
发布文章：hexo d -g 如果发布时出现错误，执行 npm install hexo-deployer-git –save

主题的更换：1 在themes文件夹下，把主题下载下来，执行git clone ………
2 打开Hexo文件夹下的_config.yml文件，修改里面的theme为主题文件夹名称。
3 hexo server 输入http://0.0.0.0:4000/会看到主题已经更改成功。
*如果长时间没写blog，会出现bash: hexo: command not found错误提示，
这个时候要找到如下路径添加到系统环境变量中去就行了。
F:\Blog\hexo\node_modules\hexo\bin

附：环境变量是在操作系统中一个具有特定名字的对象，它包含了一个或者多个应用程序所将使用到的信息。例如Windows和DOS操作系统中的path环境变量，当要求系统运行一个程序而没有告诉它程序所在的完整路径时，系统除了在当前目录下面寻找此程序外，还应到path中指定的路径去找。用户通过设置环境变量，来更好的运行进程。
环境变量一般是指在操作系统中用来指定操作系统运行环境的一些参数，比如临时文件夹位置和系统文件夹位置等。（不太理解）
