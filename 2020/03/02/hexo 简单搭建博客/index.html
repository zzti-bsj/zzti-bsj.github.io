hexo简单搭建博客

  简单爽快的记录一下搭建hexo & 部署到github上的整个过程。

1. 安装node.js & Git

  • node.js 下载地址（建议下载LTS，这个是长期支持版）
  • Git 下载地址

node.js就是运行在服务端的 JavaScript，不需要具体知道node.js是什么，除非你要做node.js开发。当然，如果好奇，参考教程如下：node.js API docs

Git & node.js下载完成之后，除了选择你要安装的路径之外，其他选项默认“下一步”安装即可。

2. 检验Git & node.js 的环境

  • 查看git版本
  • 查看node.js版本

如果没有以上的版本信息，百度搜索“配置git环境变量”和“配置node.js环境变量”，自行解决git和node.js的环境变量配置问题。

3. 安装hexo

  在安装hexo之前，为你的博客创建一个专属的文件夹，比如在E盘创建一个blog文件夹
hexo官网文档

  • 打开git安装hexo，执行：npm install -g hexo-cli
  • cd到blog文件夹下，执行：hexo init <folder>
    然后会看到如下文件：

此时说明hexo的配置初始化完成，继续下面的步骤

4. 本地测试

  • 查看本地hexo服务器，执行： hexo s
    如下图所示：

ctrl + c 结束本地服务，接下来部署到github上

5. 部署到github

  • 创建repo，格式yourgithubname.github.io，如下所示：

  • 配置本地github账户

  • 生成秘钥，ssh免密登录
    • 执行ssh-keygen -t rsa -C "youremail@example.com"
    • 如下列图示方式配置秘钥，然后复制下来：
    • 如下列图示找到settings中的ssh keys配置，把刚才复制的key添加进去。

    • 验证免密是否成功，执行ssh -T git@github.com，如图所示便是成功了
    • 打开blog文件夹，修改_config.yml文件配置（这里有个关键点，以git@github.com:YourGithubName/YourGithubName.github.io.git的形式配置repo，才可以避免每次部署时都输入用户名和密码）
//如下示例，zzti-bsj是我的github用户名
deploy:
  type: git
  repo: git@github.com:zzti-bsj/zzti-bsj.github.io.git
  branch: master
  • 最后一步，部署到github
    • hexo clean
    • hexo generate
    • hexo deploy

最后一步部署，会弹出让你输入github的用户名和密码，之后再部署的时候不会再弹出用户名和密码，因为已经使用了ssh免密。

hexo clean 是清除生成的缓存文件，如果有一天你发现怎么部署，博客上的内容还是不更新，不妨执行这个命令，清除缓存试试。

hexo generate 生成静态文件 | hexo deploy 部署，两个命令可以合并为 hexo d -g或者hexo g -d执行

这个时候访问 https://yourgithubname.github.io，可以访问到部署好的博客网站。

域名解析 & 绑定域名

  每次都访问https://zzti-bsj.github.io，一定不如访问简短的https://codeac.top方便。
  首先拥有一个域名，可以去阿里云买，或者其他的平台。

  • 登录阿里云，找到买好的域名

  • 点击域名解析，如下图添加两项



  • 最后很重要的一点，在source目录下新建一个CNAME文件，里面写上自己的域名，比如我的：codeac.top

至此，域名解析已经做完了，可能需要等上一会，再访问 https://www.codeac.top 就可以访问自己的博客了。
