程序测试模块 doctest

  • 2020-03-01
  • Bsj

doctest模块

开发高质量软件的方法之一是为每一个函数开发测试代码，并且在开发过程中经常进行测试

doctest模块提供了一个工具，扫描模块并根据程序中内嵌的文档字符串执行测试。

测试构造如同简单的将它的输出结果剪切并粘贴到文档字符串中。

通过用户提供的例子，它强化了文档，允许 doctest 模块确认代码的结果是否与文档一致

简单的测试例子

  • 正确测试案例
    >>> def average(values):
...     """Computes the arithmetic mean of a list of numbers.
...
...     >>> print(average([20, 30, 70]))
...     40.0
...     """
...     return sum(values) / len(values)
...
>>> import doctest
>>> doctest.testmod()   # 自动验证嵌入测试
TestResults(failed=0, attempted=1)
  • 错误测试案例
    >>> def average(values):
...     """Computes the arithmetic mean of a list of numbers.
...     >>> print(average([20, 30, 70]))
...     41.0
...     """
...     return sum(values) / len(values)
...
>>> import doctest
>>> doctest.testmod()

File “main“, line 3, in main.average
Failed example:
print(average([20, 30, 70]))
Expected:
41.0
Got:
40.0

1 items had failures:
1 of 1 in main.average
Test Failed 1 failures.
TestResults(failed=1, attempted=1)


&emsp;&emsp;根据以上的测试结果说明，测试的结果会返回一个`doctest.TestResults`对象
```python
>>> type(doctest.testmod())
<class 'doctest.TestResults'>
>>>

  当结果正确的时候TestResults(failed=0, attempted=1)的failed参数为0，如果测试用例的结果错误，会打印异常。
  当测试代码中没有测试构造字符串时，测试的返回结果会是TestResults(failed=0,attempted=0)
