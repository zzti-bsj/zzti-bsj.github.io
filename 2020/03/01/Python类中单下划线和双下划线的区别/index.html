Python类中单下划线和双下划线的区别

[如果有不妥的地方，欢迎提出更正]

  类是具有相同属性和方法的对象的集合，在类中的属性和方法通常都涉及权限问题。在Python中，涉及到方法和属性权限问题的标志是单下划线 和 双下划线

  如果对以下文字没有耐心，直接就去看每部分的代码实例详解

1、在单个类中

  既然类是具有相同属性和方法的对象的集合，那么我们来看看类的方法和属性的权限问题
  在c++中修饰方法和属性的权限关键字有public/protected/private，三者权限依次是递减的。private的权限最严格。
  （Python中没有这些关键字）但Python中有没有所谓的public/protected/private权限呢？理论上是有这种权限存在的，但是没有public/protected/private这三个关键字，在python中相对应的应该是无下划线、单下划线、双下划线。

类中属性/方法的权限

代码实例详解： 

# 创建一个测试类
class Test(object):
    # 测试
    def __init__(self, name, age, sex):
        """三者依次是无下划线、一个下划线、双下划线的属性"""
        self.name = name
        self._age = age
        self.__sex = sex

    # 设置三种不同的方法权限分别为public/protected/private级别
    def method_1(self):
        return "method_1", self.name, self._age, self.__sex

    def _method_2(self):
        return "method_2", self.name, self._age, self.__sex

    def __method_3(self):
        return "method_3", self.name, self._age, self.__sex


# 实例化一个Test类 做测试
test = Test('otnw', 21, '男')

# 1、类外依次访问三个属性
print(test.name)
print(test._age)  # 不合规矩，一般不直接调用protected的属性和protected方法
# print(test.__sex)
# 发现test.__sex这里会有提示，并且运行之后会报错，说明在类外不能调用类的private属性

# 2、类外依次访问三个方法
print(test.method_1())
print(test._method_2())
# print(test.__methond_3())
# 发现test.__methond_3()这里会有提示，并且运行之后会报错，说明在类外不能调用类的private方法

2、对于这些权限的属性/方法，子类继承了父类的哪些？

  涉及到继承的概念，这个词，我们在生活中常常知道儿子继承父亲的财产（父亲有什么，儿子继承什么。儿子继承父亲的财产之后，拥有了父亲的钱/公司）、太子继承皇位（皇上有什么，太子继承什么。太子继承皇位之后，拥有了皇上的江山/权利/待遇），继承的思想就是这样的思想。在继承中，子类可以看做是父类的孩子。
  所谓继承，就是子类继承父类留下来的东西。而无论是子类还是父类，而类：是具有相同属性和方法的对象的集合。 既然类只有属性和方法，那子类继承，也就只能继承父类的属性和方法咯。
  大概了解了继承之后，再看，由于不同语言有不同的规定，我们去网上找一下Python自己规定的继承的概念。

  继承：即一个派生类（derived class）继承基类（base class）的字段和方法。继承也允许把一个派生类的对象作为一个基类对象对待。例如，有这样一个设计：一个Dog类型的对象派生自Animal类，这是模拟”是一个（is-a）”关系（例图，Dog是一个Animal）。

  暂时先不对这个概念做解释，先看看就行了，没记住没关系，大概了解继承是啥就行。接下来看代码。

代码实例详解：

# 创建一个测试类
class Test(object):
    # 测试
    def __init__(self, name, age, sex):
        """三者依次是无下划线、一个下划线、双下划线的属性"""
        self.name = name
        self._age = age
        self.__sex = sex

    # 设置三种不同的方法权限分别为public/protected/private级别
    def method_1(self):
        return "method_1", self.name, self._age, self.__sex

    def _method_2(self):
        return "method_2", self.name, self._age, self.__sex

    def __method_3(self):
        return "method_3", self.name, self._age, self.__sex


# 实例化一个Test类 做测试
test = Test('otnw', 21, '男')

# 1、类外依次访问三个属性
print(test.name)
print(test._age)  # 不合规矩，一般不直接调用protected的属性和protected方法
# print(test.__sex)
# 发现test.__sex这里会有提示，并且运行之后会报错，说明在类外不能调用类的private属性

# 2、类外依次访问三个方法
# print(test.method_1())
# print(test._method_2())
# print(test.__methond_3())
# 发现test.__methond_3()这里会有提示，并且运行之后会报错，说明在类外不能调用类的private方法


# 再创建一个类，用于继承Test类
class Test1(Test):
    """
        Test类包含(public/protected/private)这三种属性和方法。

        python中是没有这三个关键字的，为了方便记忆和理解，
        后面我就使用public代表无下划线的属性/方法，
        使用protected代表单下划线的属性/方法，
        使用private代表双下划线的属性/方法。
    """
    # Test1就写个other属性吧
    def __init__(self, name, age, sex, other):
        super().__init__(name, age, sex)
        self.__other = other

    # 待会在下面的测试中依次调用以下的方法，测试是否是继承了父类的三种类型的属性
    # 使用父类public属性
    def get_attr_public(self):
        print(self.name)

    # 使用父类protected属性
    def get_attr_protected(self):
        print(self._age)

    # 使用父类private属性
    def get_attr_private(self):
        print(self.__sex)

    # 待会在下面的测试中依次调用以下的方法，测试是否是继承了父类的三种类型的方法
    # 使用父类的public方法
    def get_method_public(self):
        return self.method_1()

    # 使用父类的protected方法
    def get_method_protected(self):
        return self._method_2()

    # 使用父类的private方法
    def get_method_private(self):
        return self.__method_3()


# 创建Test1实例
test1 = Test1('bsj', 21, '男', '其他')

# 1、测试继承了父类的哪些属性
test1.get_attr_public()
test1.get_attr_protected()
# test1.get_attr_private()
# 测试结果发现，调用get_attr_private()方法会报错，没有继承父类的private属性.


# 2、测试继承了父类的哪些方法
print(test1.get_method_public())
print(test1.get_method_protected())
# print(test1.get_method_private())
# 测试结果发现，调用get_method_private()方法会报错，子类没有继承父类的private方法。

总结

类的属性和方法的权限问题

  1. 无下划线的属性可以看做 public（公共的）
  2. 单下划线的属性可以看做 protected（受保护的，虽然可以直接调用，但一般不在类外直接调用，不合规矩）
  3. 双下划线属性可以看做private（私有的）

子类继承了父类的哪些属性和方法？

  1. 子类继承了父类的无下划线/单下划线属性和方法，(public/protected)
  2. 不能继承父类的双下划线属性/方法，（private）
