### 安装django项目依赖包

* 从项目导出。生成requirements.txt  
`pip freeze > requirements.txt`

* 在服务器安装部署。执行requirements文件  
`pip install -r requirement.txt`  

* 如果不能安装成功，则手动安装每个包  


### 安装、配置mysql和redis服务
* 安装mysql服务：`sudo apt-get install redis-server`  
* 安装redis服务：`sudo apt-get install mysql-server`  

* 启动redis `service reids start`
* 创建mysql数据库  

### django部署准备工作（测试运行时不能配置这些文件）
* 修改`DEBUG`为`False`  
* 设置`ALLOW_HOST`为 `"*"`  
* `INSTALL_APPS`下添加`gunicron`  
* 搜集静态文件  
    * `python manage.py collectstatic`  
    * `STATIC_ROOT = os.path.jois(BASE_DIR, 'STATIC/')`  
    * 在static下出现admin文件夹
* 设置静态文件的路由
    * 配置urls.py文件

from django.contrib.staticfiles.urls import staticfiles_urlpatterns

……

debug为false的时候使用这个静态文件的路径

urlpatterns += staticfiles_urlpatterns()

## 一、gunicorn配置
[gunicorn配置及使用详解](https://codeac.top/2019/09/21/gunicorn%E9%85%8D%E7%BD%AE%E5%8F%8A%E4%BD%BF%E7%94%A8%E8%AF%A6%E8%A7%A3/)  

在settings同目录下新建`gunicorn.conf.py`文件，根据需要修改相关配置信息  
* 修改bind，端口号要与DjangoProject.conf的server端口相同，这里使用8000
* 根据需求修改最大并发量
* 修改访问日志和错误信息日志路径中项目名称
* 修改进程文件目录中的项目名称
* 修改proc_name为django项目名称

#!/usr/bin/env python

-- coding:utf-8 --

import os
from multiprocessing import cpu_count

bind = “0.0.0.0:8000”
daemon = True # 设置守护进程,将进程交给supervisor管理

并行工作进程数

workers = 2cpu_count() + 1 # workers是工作线程数，一般设置成：2服务器CPU个数 + 1，这样的话，在任何时候都有一半的worker在做IO

指定每个工作者的线程数

threads = 2
worker_class = “gevent”
forworded_allow_ips = ‘*’
keepalive = 6
timeout = 65
graceful_timeout = 10

设置最大并发量

worker_connections = 65535

设置进程文件目录

pidfile = ‘/www/DjangoPro/gunicorn.pid’

设置访问日志和错误信息日志路径

errorlog = ‘/www/DjangoPro/gunicorn.error.log’
accesslog = ‘/www/DjangoPro/gunicorn.access.log’

日志等级

debug : 打印全部的日志(notset等同于debug)

info : 打印info,warning,error,critical级别的日志

warning : 打印warning,error,critical级别的日志

error : 打印error,critical级别的日志

critical : 打印critical级别

loglevel = ‘info’
proc_name = ‘DjangoPro’

## 二、Nginx服务器安装、配置
[Nginx配置及使用详解](https://codeac.top/2019/09/21/Nginx配置及使用详解/)  
#### 安装nginx

$ sudo apt-get install apache2-utils # 安装apache测试命令
测试服务器对请求的承载量
$ sudo apt-get install nginx # 安装nginx服务器
$ systemctl enable nginx # 设置nginx开机自启动

* 启动nginx：`service nginx start`  
* 测试访问：输入ip，成功则出现**Welcome to nginx!**。（输入ip默认访问80端口，即访问到Nginx服务）

#### 修改默认的nginx的默认端口
&emsp;&emsp;此步骤为了让django项目使用默认端口80
* 修改Nginx默认配置文件，把Nginx服务器默认端口由80改为81或其他
    * `vi /etc/nginx/sites-enabled/default`  

#### nginx配置
* 设置并发量

vi /etc/nginx/nginx.conf
work_connections设置为65535

#### DjangoProject.conf文件配置
* 在Nginx下添加DjangoProject.conf配置文件  
`vi /etc/nginx/conf.d/djangopro.conf`
* listen使用默认的80端口
* 保持server与gunicorn中的地址相同（如果只使用本机一台服务器，gunicorn可以使用`0.0.0.0:port`）
* 注意`DjangoProject`更改为项目名称

upstream app_server {
server 127.0.0.1:8000; #与gunicorn配置中bind的地址一致
}
server {
listen 80; # niginx的端口 该端口要开放
server_name 0.0.0.0; #域名或主机地址
access_log /www/DjangoProject/host.access.log;
location = /favicon.ico {
empty_gif;
access_log off;
}
location /static/ {
root /www/DjangoProject/;
expires 30d;
access_log off;
}
location /media/ {

同上这个文件我的项目中是放图片和头像的

   root /www/DjangoProject/;

}
location / {
proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
proxy_set_header Host $http_host;
proxy_redirect off;
proxy_pass http://app_server;
}
error_page 500 502 503 504 /50x.html;
location = /50x.html {
root html;
}
}

### 启动django项目
* 重启nginx： `service nginx restart`
* 启动gunicorn：`gunicorn -w 3 -k gevent -b  DjangoProject.wsgi:application`
* 访问ip

网站部署基本完成，supervisor自启项目后续再做。
