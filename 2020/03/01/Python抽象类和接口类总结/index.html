Python接口类和抽象类总结

  总结一下接口和抽象这两个概念（不要在这个过程中想着如何写代码，去体会两者的含义）
  首先列出五个名词：抽象/抽象类/抽象方法/接口/接口类
  其中抽象和接口是面向对象的两个概念，我们重点去理解抽象类和接口类的概念，之后，抽象和接口的概念就会不点自通了。

  抽象类和接口类
  1、抽象类：含有抽象方法的类被我们称为抽象类。（含有，含有一个也是含有，只要含有抽象方法的类，就是抽象类，自然，这样说的话抽象类这个范围就囊括了接口类。）
  有的人会好奇，那么抽象类能含有普通方法吗？当然能，我的意思是只要含有抽象方法的类就是抽象类，其他爱有什么有什么。
  2、接口类：只含有抽象方法的类被我们称为接口类，“只含有抽象方法”，很明显除了抽象方法其他什么都不能有。否则它就不能被称为接口类。如果除了抽象方法还含有其他的，它就要被称为抽象类。

  抽象类：“只要含有”
  接口类：“只含有”

  我觉得上面的概念已经把抽象类和接口类解释的很清楚了，下面举例加深印象。

  再来说一个点：如果继承了含有抽象方法的类，需要在子类中把继承的抽象方法给实现（也就是重写），否则这个子类依旧是个抽象类。为什么呢？因为它继承了父类的所有属性和方法，也就是说，它继承了抽象方法，刚才我们说，含有抽象方法的类就是抽象类，所以没有实现父类抽象方法的子类就还是抽象类。

  3、抽象方法：以 @abstractmethod装饰且不实现具体功能的方法

1、代码示例详解接口类

# coding:utf-8
from abc import ABCMeta, abstractmethod


# 创建一个接口类
# 接口类AnimalRun
class AnimalRun(metaclass=ABCMeta):

    # 抽象方法
    @abstractmethod
    def run(self):
        pass


# 创建一个接口类
# 接口类AnimalSwim
class AnimalSwim(metaclass=ABCMeta):

    # 抽象方法
    @abstractmethod
    def swim(self):
        pass


# 创建一个接口类
# 接口类AnimalEat
class AnimalEat(metaclass=ABCMeta):

    # 抽象方法
    @abstractmethod
    def eat(self):
        pass


# 我们再创建一个有关人类说话这个动作的接口类
class PeopleLanguage(metaclass=ABCMeta):

    @abstractmethod
    def Language(self):
        pass


# 创建一个“飞”这个动作的接口类
class AnimalFly(metaclass=ABCMeta):

    @abstractmethod
    def fly(self):
        pass


# 创建一个植物“光合作用”这个功能的接口类
class PlantPht(metaclass=ABCMeta):

    @abstractmethod
    def pht(self):
        pass

  现在我手里有六个接口类，不要管是什么接口类，反正我现在有六个接口类。
  好了，我要开始装逼了。
  我手里有这些接口类，我现在想要造一只老虎，没错，我要造个老虎，我要看看造老虎能用到哪些接口类。
  现在来扒拉扒拉我的接口类里面有什么可以拿来让我造老虎的，众所周知，对于老虎来说，老虎没有语言，也不会光合作用，老虎会run/swim/eat，所以AnimalEat()/AnimalSwim()/AnimalRun()这些可以拿来让我造老虎。
开始造老虎，创建一个Tiger类，继承这三个接口类。
  既然继承了某个接口类，那么我就必须要实现（重写）这个接口类提供的功能。不然，我为什么要继承这个接口呢？

代码如下 

# 老虎类
class Tiger(AnimalRun, AnimalSwim, AnimalEat):
    def run(self):
        print("tiger run")

    def swim(self):
        print("tiger swim")

    def eat(self):
        print("tiger eat")


"""
造完老虎类之后，来实例化一个老虎对象。
如下，我的老虎有了run/swim/eat这些功能。

"""
tiger = Tiger()

tiger.run()
tiger.swim()
tiger.eat()

  有的同学要问了，在继承的接口类中，能不能其中一个接口类不去实现？你看，如果你不去实现这个接口类，那你为什么要继承它呢？mmp哦。
  记住一点：如果某些接口类没用，则不继承！如果继承某个接口类，就要实现某个接口类里面提供的所有方法。
就比如我们上面写的People/PlantPht这些类，老虎继承它没用，所以就不去继承。

  我觉得上面对接口类的整理已经很清晰了。

  下面来看看抽象类，刚才说了，“只要含有”抽象方法的类就是抽象类！
  下面举几个例子。 

# 抽象类举例
# 1、含有属性的抽象类
class AbsClassAttr(metaclass=ABCMeta):

    # 这里不强调什么属性，含有什么属性都行,为了方便，我们在这里使用对象属性
    def __init__(self):
        self.attr = '含有属性的抽象类'

    @abstractmethod
    def method(self):
        pass


# 抽象类无法实例化
# a = AbsClassAttr()
# a.method()
"""
当我去尝试实例化一个抽象类时，系统报错告诉我不能实例化一个含有抽象方法的类。
接下来我去创建一个类，去继承并且实例化这个抽象类中的抽象方法，且输出它的属性值。
"""


# 创建一个类继承抽象类
class ChildClass(AbsClassAttr):
    def method(self):
        print("输出继承之后的属性：" + self.attr)
        print("重写抽象类的方法")


# 实例化这个类
# 完全ojbk
c = ChildClass()
c.method()


# 2、含有普通方法的抽象类
class AbsClassMethod(metaclass=ABCMeta):

    @abstractmethod
    def method(self):
        pass

    def normal_method(self):
        print("这是一个普通方法")


# 抽象类无法实例化
# a = AbsClassMethod()
# a.method()

# 创建一个类继承抽象类
class ChildClass(AbsClassMethod):
    def method(self):
        print("重写抽象类的方法")


# 实例化这个类
# 完全ojbk
c = ChildClass()
c.method()
# 调用继承之后的普通方法
c.normal_method()


# 3、含有属性和普通方法的抽象类
class AbsClassAtrrMethod(metaclass=ABCMeta):

    def __init__(self):
        self.attr = "这是抽象类的属性"

    @abstractmethod
    def method(self):
        pass

    def normal_method(self):
        print("这是一个普通方法")


# 抽象类无法实例化
# a = AbsClassAtrrMethod()
# a.method()

# 创建一个类继承抽象类
class ChildClass(AbsClassAtrrMethod):
    def method(self):
        print("继承父类之后的属性：" + self.attr)
        print("重写抽象类的方法")


# 实例化这个类
# 完全ojbk
c = ChildClass()
c.method()
# 调用继承之后的普通方法
c.normal_method()

  以上三条测试发现，与我们在开始的时候提到的抽闲类的概念完全符合。
