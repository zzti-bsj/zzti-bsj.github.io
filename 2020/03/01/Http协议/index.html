Http协议

什么是HTTP？

  官方对http的解释：HTTP协议是Hyper Text Transfer Protocol（超文本传输协议）的缩写,是用于从万维网（WWW:World Wide Web ）服务器传输超文本到本地浏览器的传送协议。

  HTTP：超文本传输协议(HyperText Transfer Protocol)

什么是超文本？

  从字面意思来讲，文本就是普通的文本、文字，而超文本是除了文字以外的图片、html、查询结果等比文本更复杂的数据。 为了在网络中传递这些复杂的数据，官方规定了一个协议，那就是http协议

  在大概了知道了http协议是什么之后，我们深入了解一下关于http相关的知识。

http近一步解读

  1、http协议基于TCP/IP通信协议来传输数据
  2、Http的三个注意事项：
  （1）HTTP是无连接：==无连接的含义是限制每次连接只处理一个请求==。服务器处理完客户的请求，并收到客户的应答后，即断开连接。采用这种方式可以节省传输时间。
  （2）HTTP是媒体独立的：这意味着，==只要客户端和服务器知道如何处理的数据内容，任何类型的数据都可以通过HTTP发送==。客户端以及服务器指定使用适合的MIME-type内容类型。
  （3）HTTP是无状态：HTTP协议是无状态协议。==无状态是指协议对于事务处理没有记忆能力==。缺少状态意味着如果后续处理需要前面的信息，则它必须重传，这样可能导致每次连接传送的数据量增大。另一方面，在服务器不需要先前信息时它的应答就较快。

（无连接、媒体独立、无状态）

  3、http协议通信流程

下面，先来了解一下请求url的过程：
  请求url过程：我们通过浏览器访问一个url，此时浏览器发送了一个http请求给服务器，服务器接收到这个请求之后，返回响应（html页面）给浏览器，与此同时，浏览器发现该响应中（html页面）还有其他的url（网页资源文件地址，如：图片），然后又自动根据这些url去请求相应的资源文件，等到把资源全部获取，浏览器根据特定的形式，完整的把页面显示在浏览器上。
  url请求的过程总结：
（1）浏览器发送http请求
（2）服务器根据请求做出响应
（3）浏览器继续根据url自动发出http请求获取资源
（4）浏览器响应返回数据（直到没有相应的http请求发生）

  图中这个过程提到了CGI，简单了解一下：
  CGI：(Common Gateway Interface) 是 HTTP 服务器与你的或其它机器上的程序进行“交谈”的一种工具
（1）CGI是一种程序，运行在网络服务器上。
（2）CGI用来解释处理表单输入的数据，将相应信息反馈给浏览器。
（3）CGI程序使网页具有交互功能。

请求报文和响应报文

1、请求报文：
（1）组成：请求行、请求头部、空行、请求数据

2、响应：
（1）组成：状态行（返回状态码，请求是否成功）、消息报头、空行、响应正文（html）

请求方法

（1）HTTP 1.0定义了三种请求方法：GET/POST/HEAD
（2）HTTP 1.1新增了6种请求方法：PUT/DELETE/OPTIONS/CONNECT/TRACE/PATCH

  主要的是GET和POST
  （1）GET:请求数据
  （2）POST:提交数据
  （3）HEAD:类似GET，主要用于获取报头
  （4）PUT:从客户端向服务器传送数据替换指定的文档的内容。
  （5）DELETE:请求服务器删除指定的页面。

更多功能

请求头和响应头（response headers/request headers）

响应头：
  当我对一个网页检查的时候，我选择一个文件，并点击查看。

  如上图，我点击了一个js文件，从这个图中，我得知它的请求头和响应头的相关信息，其中响应头中的content-type是text/javascript。
  更多content-type对应文件类型
  除此之外还有其他的响应头信息：date/last-modified/server/content-encoding 等等

请求头：
如下图所示，我们可以看到一些请求头里的信息：

其中比较常用的有：
（1）User-Agent：用户代理
（2）Cookie：浏览器记录的用户信息
（3）Host：服务器主机地址（如百度的是www.baidu.com）
（4）Accept：可接受的网页格式
（5）Accept-Encoding：可接受的编码
（6）Accept-Language：可接受的语言

拓展：Cookie（浏览器中寄存的小型数据体，它可以记载和服务器相关的用户信息）常含有四个字段：
（1）name：字段名
（2）value：值
（3）domain：作用的域名
（4）expires：失效时间

状态码

  服务器返回的响应头中有个状态码，该状态码对应着服务器的响应状态。

  HTTP状态码由三个十进制数字组成，第一个十进制数字定义了状态码的类型，后两个数字没有分类的作用。HTTP状态码共分为5种类型：

  HTTP状态码分类
分类 分类描述
1** 信息，服务器收到请求，需要请求者继续执行操作
2** 成功，操作被成功接收并处理
3** 重定向，需要进一步的操作以完成请求
4** 客户端错误，请求包含语法错误或无法完成请求
5** 服务器错误，服务器在处理请求的过程中发生了错误

  常见的状态码
200 - 请求成功
301 - 资源（网页等）被永久转移到其它URL
404 - 请求的资源（网页等）不存在
405 Method Not Allowed 客户端请求中的方法被禁止
500 - 内部服务器错误
504 Gateway Time-out 充当网关或代理的服务器，未及时从远端服务器获取请求
