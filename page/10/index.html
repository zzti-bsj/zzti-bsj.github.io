1.scrapyd文档

一、scrapyd文档概述

scrapy简介

  Scrapyd是一个运行和部署爬虫的应用，它能够使你部署你的爬虫项目，并且使用一个JSON API控制爬虫项目。

项目和版本

  Scrapyd能够部署多个文件，每个文件有多个版本，但是只有最新的版本才能够发起爬虫。也就是说，同一个爬虫，可以被覆盖。

scrapyd如何工作

  • 典型的守护进程
  • 为每个爬虫项目提供监听服务，生成进程
  • 同时运行多个进程
  • scrapyd在Twisted里面执行
  • 提供json网络服务去部署和调度爬虫

开启Scrapyd服务器

scrapyd

Scrapyd的网络接口

scrapyd自带一个网络接口，为了运行爬虫和访问日志 Scrapyd的Web接口

依赖版本

  • python3.6
  • scrapy==1.6.0
  • scrapyd==1.2.0
  • scrapyd-client==1.2.0a1
  • Twisted==17.1.0 (17和18版本应该都可以)

Scrapyd的默认配置文件

  Scrapyd的默认配置文件是Scrapyd依赖包下的default_scrapyd.conf
  远程部署Scrapyd时需要修改此文件配置。

2.curl管理Scrapyd服务器

curl管理Scrapyd服务器

检查scrapyd服务状态(daemonstatus.json)

curl http://localhost:6800/daemonstatus.json

  • 请求类型：GET

正常会返回类似如下的json数据
{ "status": "ok", "running": "0", "pending": "0", "finished": "0", "node_name": "node-name" }

增加项目版本(addversion.json)

  添加一个项目版本，如果版本不存在，则创建项目。

curl http://localhost:6800/addversion.json -F project=myproject -F version=r23 -F egg=@myproject.egg

  • 请求类型： POST
  • 参数
    • project：项目名称
    • version：项目版本
    • egg：包含项目代码的实例

调度爬虫项目运行(schedule.json)

  调度爬虫项目运行，返回项目id

curl http://localhost:6800/schedule.json -d project=myproject -d spider=somespider -d setting=DOWNLOAD_DELAY=2

  • 请求类型：POST
  • 参数
    • project：项目名称
    • spider：爬虫名称
    • settings：爬虫运行时的配置

停止爬虫项目(cancle.json)

  如果爬虫项目在等待，则移除此爬虫。如果项目已经运行，则停止该爬虫运行。

curl http://localhost:6800/cancel.json -d project=myproject -d job=6487ec79947edab326d6db28a2d86511e8247444

  • 请求类型：POST
  • 参数
    • project：项目名称
    • job：job id

获取服务器上的爬虫项目(listprojects.json)

curl http://localhost:6800/listprojects.json

  • 请求类型：GET

获取项目版本(listversions.json)

  获取Scrapyd服务器上某些项目的版本
curl http://localhost:6800/listversions.json?project=myproject

  • 请求类型：GET
  • 参数
    • project：项目名称

获取可用的爬虫项目(listspiders.json)

  罗列Scrapyd服务器上最新版本的爬虫项目。
curl http://localhost:6800/listspiders.json?project=myproject

  • 请求类型：GET
  • 参数
    • project：项目名称
    • _version：需要查看的项目的版本

获取jobs(listjobs.json)

  获取等待、运行、完成的jobs。

curl http://localhost:6800/listjobs.json?project=myproject | python -m json.tool

  • 请求类型：GET
  • 参数
    • project：项目名称，这个参数限制只查询当前项目的爬虫的状态

删除项目版本

  删除一个项目版本。

curl http://localhost:6800/delversion.json -d project=myproject -d version=r99

  • 请求类型： POST
  • 参数
    • project：项目名称
    • version：项目版本

删除一个项目的所有版本

curl http://localhost:6800/delproject.json -d project=myproject

  • 请求类型： POST
  • 参数
    • project：项目名称

进步、改变自己：从改变工作模式开始

先前的工作模式

  我觉得自己先前的工作模式是一种非常浪费时间、进步很慢的方式！

  “我有一种患得患失的恐惧感，我总是想造自己的轮子，它让我不能更好的使用别人造好的轮子，不能更好的进步。” 有时候我会特别抗拒拿来就用的模块，可能那就是很好的一个模块，但是代码终归不是自己写的，思想上特别抗拒。但如果抗拒使用别人造好的轮子，这很可怕，这无疑是一个非常浪费开发时间的行为！

  “为什么每次都想造自己的轮子？因为我怕失去造轮子的能力。” 我是一名开发者，我更是一个学者。我不能仅仅局限于具有使用工具的能力，我必须还要拥有能够造工具的能力。对于解决可变性问题的能力，我是永远不能够丢失掉的。但如果不断重复的造别人造过的轮子，这也很可怕，我将会失去大量的学习时间。

  “开发的时候想使用新技术，学习的时候想做开发练手。” 之前的学习模式是一种混乱的模式，我没有清楚的认识到自己是在开发还是在学习，总是孤军奋战，也不太喜欢团队合作。开发的时候想要使用新技术，学习的时候想要做开发练手，这也是很可怕的一件事情。

优化工作模式

一、分清学习时间和开发时间

  “学习是自我输入，开发是自我输出。” 学习和开发的关系，就像计算机IO流一样，输出的同时，过多的输入会占用输出的时间，降低输出的效率。同理，输入的同时，也不要过多的输出。

  “开发时间要做什么？开发是自我输出，这个时候过多的自我输入会降低开发效率” 毫无疑问，开发的时候，讲究开发的效率，需要很快的开发出完整的项目，需要的是自身现有的知识、信息检索的能力以及使用别人轮子的能力。现有的知识越是丰富，开发的过程就愈发的快。信息检索能够精准的、友好的检索到问题的解决方案以及更好的开发方案，所以也需要更多信息检索以及能够提供技术和信息的平台，这些技术平台需要自己去积累。更重要的是使用别人造的轮子的能力，对于别人造的轮子，能够快速的理解并且能够使用。

  “学习时间要做什么？如何学习框架和学习不易更迭的技术” 学习有很多种。学习不易更迭或者不会更迭的技术，需要由浅入深的学，需要花费时间和精力总结。而学习框架和“别人封装的代码” ，只需要学习其架构和运行机制，在开发能力的基本功扎实的基础上，这只是一种记忆性的学习行为，很好的记忆方式就能够满足这种学习。

二、怎么学习别人的轮子、使用别人的轮子

  “怎么学习别人的轮子、使用别人的轮子？” 众所周知，站在巨人的肩上，才能够看的更远。在紧张的开发阶段，需要快速的拿来别人的轮子就用，可能需要经常从网上找一些好的代码修改，这个时候要能够快速转化成自己的东西。开发结束之后才是学习别人轮子的阶段，这个时候要花费一点点时间，去理解这个东西的架构和运行机制。
  “用别人的轮子，造别人没造过的轮子” 学会学习和使用别人的轮子，转化所有能够想到的知识，造别人没造过的轮子。
