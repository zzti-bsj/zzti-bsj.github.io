简单了解常见的加密方式

几种常见的加密方式

md5（Message Digest Algorithm）

  md5，中文名称为消息摘要算法，字面意思上来说，是一种采用摘要进行加密的算法。
  md5加密的特点：
  1.加密之后是一个32位的字符串
  2.属于不可逆加密方式
注：md5算法虽然不可逆，但是对于一些简单的字符串，还是可以通过md5加密之后暴力匹配出来的。
（网上有一些专门破解md5加密的网站，有兴趣的可以搜索一下，下面给一段代码，可以生成简单的md5加密之后的数据，贴在解密网站上是可以解密的。）
  代码示例：

AES（Advanced Encryption Standard）

  AES：高级加密标准，DES是它的前任标准。

DES（Data Encryption Standard）

  DES：数据加密标准

哈希+盐

  哈希+盐，也是一种常用的加密手段。哈希，是利用加密方法，盐：一般是随机字符串。通过哈希+盐的方式进行加密，使加密的安全性更高。
  比如说，md5可以通过暴力碰撞的方式进行破解，那密码的安全性就受到了威胁。在这种情况下，我们把原字符串+盐，然厚进行加密，就得到了一个新的md5值，这样的话，提升了加密的安全性。

了解json

JSON（JavaScript Object Notation）JavaScript 对象表示法
json是纯文本

json是什么？

1、JSON是存储和交换文本信息的语法，类似于xml
2、JSON是轻量级的文本数据交换格式

相比于xml，json更小、更快、更容易解析

json的语法

在json的语法格式中：
（1）用{ }表示JavaScript的对象。对象中只能包含键值对，键必须使用双引号包括，值可以是JavaScript的对象、数组、JavaScript的基本数据类型的数据（整型/浮点型/布尔型/null）
（2）用[ ]表示JavaScript的数组。数组中可以包含对象、数组、javascript的基本数据类型的数据（整型/浮点型/布尔型/null）

  在python中，json的bool类型/null将被转成python中对应的bool类型/None

scrapy运行机制和项目架构

scrapy

  scrapy是一个爬虫框架。
  在一个框架中，首先要了解它的运行机制和框架的项目架构

scrapy的运行机制

  

  上图是scrapy的运行机制流程图

Scheduler：调度器
Downloader：下载器
Spider：爬虫文件（爬虫）
Scrapy Engine：项目的引擎
Item/Pipelines：items/管道

这个流程图的具体执行流程：
（1）爬虫发出爬取请求，引擎从调度器队列中取出url，送到下载器 (爬虫->引擎->下载器)
（2）下载器收到请求并下载内容，返回给引擎，引擎将内容返回给爬虫 (下载器->引擎->爬虫)
（3）爬虫解析从引擎获取的内容，将解析后的数据项给管道处理 (爬虫->管道)
（4）管道负责将传来的数据做相应的处理，如：下载/存储 (管道处理数据)

scrapy的项目架构


上图是一个新建的scrapy项目的整体架构

（1）items.py：items中的类用来定义数据项，其实例结构形如字典，用来暂存数据，后续交给管道处理。
（2）pipelines.py：中文名称，管道。接收数据项item，在管道文件中编写处理数据的相关类和方法。
（3）middlewares.py：中文名称，中间件。处理请求和相应的相关事务。 目测有三种中间件：
  spidermiddlewares
  schedulermiddlewares
  downloadermiddlewares，
  其中最为常用的是downloadermiddlewares中间件。
（4）settings.py：scrapy项目的配置文件，这里是核心配置文件。基本架构相关的配置都在其中。
  比如：配置scrapy调用相关ITEM_PIPELINES/下载时使用的管道DOWNLOADER_MIDDLEWARES

bloomfilter4py3模块：python实现布隆过滤

布隆过滤

#布隆过滤/数据去重

  简单介绍一下布隆过滤：
  布隆过滤器（Bloom Filter）是1970年由布隆提出的。它实际上是一个很长的二进制向量和一系列随机映射函数。布隆过滤器可以用于检索一个元素是否在一个集合中。它的优点是空间效率和查询时间都比一般的算法要好的多，缺点是有一定的误识别率和删除困难
优点：
（1）空间效率高
（2）查询时间短
缺点：
（1）有一定的误识别率
（2）删除困难

bloomfilter4py3模块

  bloomfilter4py3是Python实现布隆过滤的一个模块，这里我使用这个模块实现布隆过滤涉及的一系列数据去重问题
bloomfilter4py3
  1、安装：
pip install bloomfilter4py3
  2、引入：
import bloomfilter

使用总结

bloomfilter4py3常用的有三个方法

test()

参数：
item：需要去重判断的数据，如果存在，test()返回true，否则返回false

add()

参数：
item：需要与bit位映射的数据，每次add之后需要保存一下。

save()

参数：
filepath：保存布隆过滤文件的文件路径

Bloomfilter()

构造方法Bloomfilter()
参数：
nbits_filepath：布隆过滤文件路径或者bit数，一个bit可以映射一条数据

布隆过滤实例

  为了测试这个模块对bit位的利用率，我在Bloomfilter()方法中设置了100个bit，并且从我本地数据库中取出100条数据，按顺序加入，我发现其利用率差不多是50%，空间开辟的小了，冲突率很高。
  在使用bloomfilter的时候，尽量把参数nbits设置的大一些
  1024 -> 1kb -> 可最多映射1024个数据
