以多任务的角度理解进程和线程

概述

  网上翻了这么多，都是对官方进程和线程的理解，但是很多人大不是为了想要知道官方对线程和进程的理解，而是一个更能具体的通透的理解。
  本篇文章以实例，以多任务为切入点，简单的理解下进程/多进程和线程/多线程，争取跳出进程和线程的坑。

多任务

  什么是多任务？
  一台电脑同时听音乐、上网、编写word、打游戏，这就是多任务。

进程/多进程

  进程：听音乐是进程、上网是进程、编写word也是进程、打游戏也是进程。无论是单核还是多核的CPU，都能够实现多进程（多任务）。
  多进程：由于CPU的执行速度非常非常快！！！单核CPU通过轮流执行任务，实现多个进程同时执行。比如：听音乐0.01秒，然后word码字0.01秒，然后浏览器刷新内容0.01秒；

线程/多线程

  线程：编写word时，可以编写、修改、保存；运行网易云可以听音乐、搜索音乐；在一个进程中可以做多个不同的任务，这些任务称为线程。
  这是一个非常非常具有代表性的例子：
  多线程（播放电影）：播放一个电影是一个进程，播放电影需要同时播放音频和视频，这是多线程任务；如果等视频播放完毕之后再播放音频，这将是单线程任务，单线程任务使线程按顺序执行；

多进程和多线程实现方式

  多进程和多线程实现方式都是通过操作系统切换任务实现的，执行某个进程（线程）某一段时间（比如0.0.1秒），然后执行下一个任务同样的时间。

单线程的进程实现多任务的三种方法

  一个进程可以拥有多个线程，一个进程至少有一个线程。如果一个进程只有一个线程，如何实现多任务呢？

  • 多进程
  • 一个进程，多个线程
  • 多个进程+多个线程

Python无法实现多线程并发

Python没有真正的多线程并发

  启动与CPU核心数量相同的N个线程，在4核CPU上可以监控到CPU占用率仅有160%，也就是使用不到两核。

  即使启动100个线程，使用率也就170%左右，仍然不到两核。

  但是用C、C++或Java来改写相同的死循环，直接可以把全部核心跑满，4核就跑到400%，8核就跑到800%，为什么Python不行呢？

  因为Python的线程虽然是真正的线程，但解释器执行代码时，有一个GIL锁：Global Interpreter Lock，任何Python线程执行前，必须先获得GIL锁，然后，每执行100条字节码，解释器就自动释放GIL锁，让别的线程有机会执行。这个GIL全局锁实际上把所有线程的执行代码都给上了锁，所以，多线程在Python中只能交替执行，即使100个线程跑在100核CPU上，也只能用到1个核。

能实现多线程，不能实现多线程并发（纠正误区）

  Python可以实现多线程，但是无法实现多线程并发的任务；多线程并发只是Python的一个美丽的梦。

Python多进程并发

  Python的多进程可以实现多任务并发，每一个进程都有各自独立的GIL，互不影响。

python 协程

协程(单进程的异步编程模型、单线程)

  协程：Coroutine
  子程序调用：递归时经常会调用自己，中断而去执行更深层的函数，结束最底层的函数执行之后层层返回往下接着执行！
  协程：协程不同于子程序调用，协程没有经过函数调用。比如在执行A()一段时间后，突然切换去执行B()，这样看着像是多线程，分别执行A()、B()，但实际上只使用一个线程执行。

协程优势

  以生产者消费者问题为例，一个线程负责生产，另一个线程负责消费，这个时候涉及到线程锁，以及线程切换，极大的浪费了时间；而使用协程，因为是单线程，所以不需要线程切换，省去了切换线程花费的时间，并且不涉及到数据互斥锁问题；
  在一个问题中，线程的数量越多，使用协程的优势就越明显。
