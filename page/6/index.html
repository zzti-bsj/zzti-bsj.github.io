ubuntu16.04部署django项目

ubuntu16.04部署django项目

  • ubuntu16.04
  • nginx
  • django==2.0
  • python3.6

更改ubuntu16.04下载源

  • 备份之前的源
    sudo cp /etc/apt/sources.list /etc/apt/source_init.list

  • 删除之前的源 sudo rm /etc/apt/sources.list

  • 配置新的源：sudo vi /etc/apt/sources.list
      这里提供一些源，无法满足需要则百度

  • 更新源：sudo apt-get update

    安装python3.6

  • 安装python3.6

  • 找到python3目录，把Python3的其他版本替换成python3.6
    which python3

  • 删除快捷键。
    rm python3

  • 创建软链接
    ln -s python3.6 python3

  • 安装对应的python3的pip
    apt-get install -y python3-pip

  • 更改python对应python3

  • 后面的数字100/150是优先级

  • 升级pip
    pip install --upgrade pip 或者
    pip3 install --upgrade pip

  • 如果找不到pip或者pip3：
  • 重新安装 pip 但不是通过 apt-get 而是通过 python -m
    • 执行：sudo python -m pip install --upgrade --force-reinstall pip或者sudo python3 -m pip install --upgrade --force-reinstall pip
  • 参考以下博客：
    https://blog.csdn.net/cjeric/article/details/71104445
    https://www.cnblogs.com/banshaohuan/p/10963547.html?tdsourcetag=s_pctim_aiomsg

安装django项目依赖包

  • 从项目导出。生成requirements.txt
    pip freeze > requirements.txt

  • 在服务器安装部署。执行requirements文件
    pip install -r requirement.txt

  • 如果不能安装成功，则手动安装每个包

安装、配置mysql和redis服务

  • 安装mysql服务：sudo apt-get install redis-server

  • 安装redis服务：sudo apt-get install mysql-server

  • 启动redis service reids start

  • 创建mysql数据库

django部署准备工作（测试运行时不能配置这些文件）

  • 修改DEBUG为False
  • 设置ALLOW_HOST为 "*"
  • INSTALL_APPS下添加gunicron
  • 搜集静态文件
    • python manage.py collectstatic
    • STATIC_ROOT = os.path.jois(BASE_DIR, 'STATIC/')
    • 在static下出现admin文件夹
  • 设置静态文件的路由
    • 配置urls.py文件
  • 数据迁移。
    • 服务器上没有数据库文件的时候，删除本地历史数据迁移文件，在ubuntu16.04服务器上执行如下命令：
    • python manage.py makemigrations
    • python manage.py migrate

安装支持django项目运行的依赖包

此时在项目根目录下可以测试项目运行

DjangoProject: 项目名称
gunicorn -w 3 -k gevent -b 0.0.0.0:8000 DjangoProject.wsgi:application

一、gunicorn配置

gunicorn配置及使用详解

在settings同目录下新建gunicorn.conf.py文件，根据需要修改相关配置信息

  • 修改bind，端口号要与DjangoProject.conf的server端口相同，这里使用8000
  • 根据需求修改最大并发量
  • 修改访问日志和错误信息日志路径中项目名称
  • 修改进程文件目录中的项目名称
  • 修改proc_name为django项目名称

    二、Nginx服务器安装、配置

    Nginx配置及使用详解

    安装nginx

  • 启动nginx：service nginx start
  • 测试访问：输入ip，成功则出现Welcome to nginx!。（输入ip默认访问80端口，即访问到Nginx服务）

修改默认的nginx的默认端口

  此步骤为了让django项目使用默认端口80

  • 修改Nginx默认配置文件，把Nginx服务器默认端口由80改为81或其他
    • vi /etc/nginx/sites-enabled/default

nginx配置

  • 设置并发量

    DjangoProject.conf文件配置

  • 在Nginx下添加DjangoProject.conf配置文件
    vi /etc/nginx/conf.d/djangopro.conf
  • listen使用默认的80端口
  • 保持server与gunicorn中的地址相同（如果只使用本机一台服务器，gunicorn可以使用0.0.0.0:port）
  • 注意DjangoProject更改为项目名称

    启动django项目

  • 重启nginx： service nginx restart
  • 启动gunicorn：gunicorn -w 3 -k gevent -b DjangoProject.wsgi:application
  • 访问ip

网站部署基本完成，supervisor自启项目后续再做。

浏览器如何判断请求是post还是get

浏览器默认请求

  • 浏览器的默认请求是get

    post请求

  • ajax发送请求设置type为post
  • form表单提供method设置请求类型

    对get/post返回响应

  • 服务器对get请求返回响应
  • 服务器收到post请求先处理数据，再返回响应（根据服务器上的项目处理相应的数据）

    请求和响应

  • request：发送到服务器，作为处理响应的特定方法的参数。
  • response：返回到显示数据的页面或者返回json数据，显示给用户。

CSRF跨站请求伪造攻击

CSRF跨站请求伪造攻击

  CSRF攻击是通过跨域请求的恶意攻击网站的方式。
  CSRF（Cross-site request forgery）跨站请求伪造。指在浏览器打开A网站的同时，本地保存着A的cookie，并且保持着与A的session对话。此时打开恶意网站B，在B网站中拥有A网站的某个url，此时点击此url，在A网站不知情的情况通过跨域恶意攻击A网站。

机器学习（二）剖析一个新问题

一、剖析一个新问题

1、属性和标签

  1、数据可以使用特征（变量）矩阵（表格）来表示，行代表实例，列代表属性。
  2、矩阵的列分为属性（独立变量）、标签（依赖变量）

  • 属性：用来做预测的数据
  • 标签：是被预测的数据

2、回归问题和分类问题

  当标签为数值的时候，预测问题成为回归问题。
  当标签为类别的时候，预测问题为分类问题。

  数值 -> 回归问题
  类别 -> 分类问题

二、属性和标签的不同类型决定模型的选择

属性的两种类型：

  • 数值变量
  • 类别变量

同理，标签也能分为数值标签和类别标签，其中二值化的类别数值可以使用0 1 代替表示

属性和标签的不同类型决定了模型是回归模型还是分类模型

三、分类问题可能比回归问题简单

  比如：一个地形图，有一个等高线20米。而另一个地形图，有很多等高线。第一个地形图只需要划分出高于20和低于20的类别，一个分类器只画出一个等高线。而第二个地形图则需要划分不同的，不需要考虑远近距离之类的问题，而回归方法则需要绘制出一个完整的地形图。

四、新数据的注意事项

新数据需要注意的事项：

  • 数据的行、列
  • 类别变量的数目、取值范围
  • 缺失的值
  • 属性和标签的统计特性

对于缺失值的处理方案：
1、缺失值的实例过多，总体实例少且数据稀有：补充缺失的值，一种方法是取每个缺失的值对应的所有列的平均值。 这种方法称为遗失值插补
2、缺失的值廉价，如网上爬取的数据：如果可以丢弃，可以选择丢弃。
